# 测试执行计划

## 第一阶段：环境搭建与 Mock

- **目标**: 启用基于浏览器的测试，Mock 缺失的 Tauri API。
- **行动**: 修改 `src/renderer/api/tauriAPI.ts`，在非 Tauri 环境下回退到 `electronAPI-mock.ts`。
- **验证**: 浏览器 Agent 访问 `localhost:3000`，确认无控制台错误。

## 第二阶段：功能验证 (Mock环境)

使用浏览器 Agent 执行以下测试用例：

### 第一批：核心生命周期

1.  **TC-001 (加载)**: 访问页面。
2.  **TC-003 (解锁)**: 使用默认 Mock 密码（如果有）或绕过逻辑。

### 第二批：密码增删改查 (CRUD)

3.  **TC-006 (添加)**: 创建一个虚拟密码条目。
4.  **TC-005 (列表)**: 验证条目出现在列表中。
5.  **TC-009 (搜索)**: 搜索刚刚创建的条目。
6.  **TC-008 (删除)**: 删除该条目。

## 第三阶段：问题修复

- 一旦发现 Bug（例如 UI Key 缺失、布局错乱、逻辑错误），立即在 React 代码中修复。

## 第四阶段：原生测试 (未来)

- 待浏览器端逻辑稳定后，解决 `cargo tauri dev` 报错，进行原生后端集成测试。
